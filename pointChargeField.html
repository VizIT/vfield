Field Lines From a Point Charge

 

Field lines are another common way of visualizing vector fields. This time we show field lines emanating from the same positive charge as the first example. We also draw the charge itself. 


<script defer src=http://vizit.github.io/vfield/js/VField.min.js>
</script>

Include the same VField package on your page. 


<script type="text/javascript">
  // This object is loaded by the VizBuilder on the DOMReady event.
  var VISUALIZATION_CONFIG
      = {
          // The electric field graph displays charges,
          // field lines and gaussian surfaces.
          type:      "electric field",
          // The id of the canvas we draw into
          canvas:    "chargeCanvas",
          arrowHeadSize: 1.0,
          arrowSpacing: 15,
          scale:     50.0,
          // Position a single charge at the origin
          elements:
            {
              type: "charge",  charge:  5.0,
              x: 0.0, y: 0.0, z: 0.0,
              fieldLineDensity: 5.0, name: "charge1"
            }
        };
</script>

The configuration, though, is different. This visualization is an electric field diagram. The electric field diagrams draw the charges and the resulting field lines. An interesting difference is the use of fieldLineDensity to determine the number of field lines that are drawn. This is because field line diagrams indicate the strength of the field by changing the field line density. So that increasing the charge increases the number of field lines. We will see how to dynamically change the charge in a later example. The visualization is drawn on a canvas with the given id chargeCanvas. 


<figure class="center">
  <!--This is the canvas for our visualization. -->
  <canvas id="chargeCanvas" width="300" height="300"></canvas>
  <figcaption>The electric field from a point charge represented
              as a set of field lines.</figcaption>
</figure>

Give a place to draw the visualization. The canvas with an id matching the canvas id given in the visualization configuration is the only requirement. Wrapping it with a figure element gives a more polished appearance for the visualization and the caption.
